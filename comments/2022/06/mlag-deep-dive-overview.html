    Regarding "It could also try to restart LACP using a different system ID hoping that the remote nodes don’t get confused, but I have yet to see an implementation that would go that far."

    The Cumulus Linux MLAG implementation is documented to do this in some peer link failure scenarios:

    • Scenario 1: Peer Link Failure, clagd-backup-ip Is Active
    • Scenario 2: Peer Link Failure, clagd-backup-ip Is not Active
    • Scenario 4: Primary Power Off, clagd-backup-ip Is not Active
