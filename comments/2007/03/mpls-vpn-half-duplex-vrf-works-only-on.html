  2. Ivan Pepelnjak 19 March 2007 10:22
    There are a few variants of the basic hub-and-spoke approach, but more-or-less they all require a VRF per spoke site. The original architecture is described in the MPLS VPN Architectures book and can be simplified if you can use default routing within VPN. Further enhancements are possible with VRF address selection or VRF selection with policy-based routing, but both options are pretty complex.

    I'll write an article on these options in my IP Corner column and post a comment here when it's available.
