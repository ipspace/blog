  4. Claer 17 September 2020 07:02

    The idea for the host to control its own security is bad as mentioned by Steffann

    What have been done in the past are distributed host agents firewalls. Here are the benefits: - Host has visibility on user connected and program launched. You can define your rules based on programm name rather than classical UDP/TCP/IP tuple. - Distributed Load. Its easier to distribute the load when you apply the filtering policies on the hosts directly - Traffic filtering on the same subnet is possible.

    As always, there are drawbacks. - Manage another software agent can be problematic (think different OS versions like RHEL 6, RHEL 8 and what to do with that business critical FreeBSD who has no agent available?) - Policies are harder to define and manage - IP filtering at the subnet level does not immune you to L2 attacks.
