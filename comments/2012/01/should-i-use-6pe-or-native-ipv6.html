  6. Ivan Pepelnjak 06 January 2012 06:02
    IPv6 and MPLS don't mix well (yet). There is no LDP for IPv6, so you can't map an IPv6 FEC to MPLS LSP. Also, because MPLS TE relies exclusively on IPv4, it cannot modify the IPv6 forwarding table. Actually, IS-IS tries to do just that and fails miserably.

    You can run IPv4 over MPLS/TE/FRR, and native IPv6 in parallel, which means that IPv6 won't enjoy the protection offered by MPLS FRR. If that's OK, go ahead. If you want very FRR-based convergence for IPv6, use 6PE.

    LFA for IPv6 is definitely possible, but not implemented (AFAIK).
