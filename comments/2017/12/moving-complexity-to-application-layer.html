    It would be better if OS APIs were providing a way to connect to a DNS name directly instead of having to connect to a single IP, then it would be up to the OS to try, or not, to reach every IP that are resolved for the particular DNS name. Then it could benefit to all application. If we move complexity to application then it means every application owner have to handle such complexity while OSI Layers are here to provide service to upper layer.
    TCP provide reliable connection to all application using it, application developer doesn't have to take care of reliability of packets, so another "common service" provided by OS could be to avoid the connectivity failure by trying all IPs of a DNS entry.
