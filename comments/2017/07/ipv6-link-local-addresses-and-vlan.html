  4. Gary Nigg 13 July 2017 01:09
    I think much confusion comes from the distinction between "ports" and "interfaces." To me, there is a significant difference between them. "Ports" are L2 forwarding entities where as "interfaces" are L3 forwarding entities (which contain IP addresses). This is why pretty much all 802.3 & 802.1 IEEE stds use "ports." But because Cisco first created routers back in the day, their CLI often treats them as synonyms (unfortunately). A VLAN interface can consist of multiple L2 ports. Spanning Tree can block 3 of 4 ports on a VLAN interface. If STP respans, you don't want an IP host to have to re-ARP. So as you said, an *interface* must be represented by a single MAC address for things like ARP. But, L2 protocols like STP, LACP, LLDP need to distinguish among a set of ports in the same broadcast domain to determine best paths, LAG members, and physical connections.
