    In enterprise environments, especially when utilizing Juniper devices, it's actually more common to use RSVP-only based LSPs and not to run LDP at all. Works just fine, if you don't need to add new PE-routers too often. And if you have an NMS with provisioning possibility, it doesn't even matter if you add more routers every now and then. Of course the network will require a lot more state information which has to be soft-refreshed every now and then, but if we are talking about <100 PE-routers, it's not a problem with current hardware having a lot of CPU and ASIC-based processing power.
