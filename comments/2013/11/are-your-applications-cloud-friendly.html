      How about trying to make the applications as OPEN and PORTABLE as possible?
      Regarding the firewall issues, you could use the same process to discover port numbers as applications should, i.e ... DNS(Sec)!
      I'm not sure if it's possible with the security products which are available right now on the market, but it would be so much simpler to allow/ban applications (with their port numbers dynamically discovered through DNS) on Firewalls instead of manually configuring their port numbers inside ACLs or any other tools...
      Regarding the IP address, the application SHOULD NOT know anything about it; if it needs it, there are APIs that can easily return that dynamic information from lower IPv4/IPv6 layers.
      As a general rule, applications and network/transport layers SHOULD always be decoupled.
