    There is a similar case I know of where 802.1X enabled Cisco Catalyst switches do that for IP Device Tracking. Some colleagues were complaining due to not receiving an address via DHCP ("Duplicate IP Address 0.0.0.0" error). Workarounds were to have SVI in each user VLAN for probe sourcing or to increase probe timers.
