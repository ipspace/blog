    I verified this on actual hardware on 17.18.01. Previous versions consolidate the two lines into one with the "both" keyword at the end. Now the commands override each other. If you enter standard first and then extended, extended is all you get. If you go the other way, standard will override extended. (Interestingly enough, even if you enter the "standard" keyword, it shows up in the config simply as "neighbor x.x.x.x send-community". I'm not a huge fan of re-working the user-input syntax, but these commands are as old as time.) You can still enter "neighbor x.x.x.x send-community both".

    I don't know why we changed this. I'm pretty sure it's worked this way for as long as I've been configuring BGP, and that's a long time. I'll see if I can find anything out.
