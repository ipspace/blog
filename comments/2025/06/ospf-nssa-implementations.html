    The problem is that IETF does not use the Protocol Implementation Compliance Statements (PICS) in general. Each protocol standard should be defined by ASN.1 and should provide a TTCN-3 test set generated from the ASN.1 specification. The protocol standard should also provide a PICS Proforma. The procurers should ask for the PICS from each vendor. The PICS should be supported by test evidence, not just a statement. Potential deviations should be explained and justified.

    That is what you get when you opt for cheap quick and dirty, instead of a proper disciplined engineering design...
