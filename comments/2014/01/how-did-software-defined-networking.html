  3. Jeff Behrns 15 January 2014 06:53

    As an early-mid career network engineer I hate the term "software defined networking" because it implies that the software folks will be ruling / controlling the network. OpenFlow is an application that runs on top of a pre-existing network. The chicken can't define the egg from which it hatched. "SDN" gives software guys the idea that they can put on a network hat and take over the job of us slow network engineers. Most network engineers I've met were former application programmers who entered networking for the challenge (i.e. C/C++/Java got boring). If only we could create an NDS movement - Network Defined Software :) And haven't all networks from the beginning of silicon time been software defined; the software being IOS, Baystack, NIC drivers, etc?

    Personally I don't want my router/switch to be bossed around by a dictator controller no matter who is doing the central programming. If the controller gets hacked or fails your whole network is dead. I want my router/switch to be able to think independently and react appropriately to changes in network conditions.
