  3. Bhargav 31 January 2014 06:03
    Lately, what i have been undestanding from various virutalization solution is that there is no concept of L2 or L3 tables like what one observes in physical world. There is either just one table (route table) or max two tables (ARP table and route table) for both inter-subnet and intra-subnet. This means that intra-subnet is also subject to routing lookup. As every thing is done is software, each vendor implement different tricks. For example, HNV does a routing lookup for intra-subnet but maintains the semantics of Tradiitional L2 like no-mac rewrite, no TTL reduction etc. Open-contrail uses vRouter for intra-subnet & inter-subnet but does not reduce the TTL at all. I believe NSX DRS uses both DRS & VDS which is similar to traditional routing-switching model
