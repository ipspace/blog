    A concern that I would have with this configuration is what would happen if the box reloads between 17:01 and 07:59. The default state would mean that the route would be up. As the EEM is event-basedd, no event would be triggered until the next morning. Routing will then happen over the tunnel during a time frame where it shouldn't. Something like this might be a more robust solution:

    access-list 100 permit ip any any time-range time1
    !
    route-map default permit 10
    match ip address 100
    set interface Tunnel0
    !
    !
    time-range time1
    periodic daily 8:00 to 17:00
    !
    interface FastEthernet0/0
    ip policy route-map default
