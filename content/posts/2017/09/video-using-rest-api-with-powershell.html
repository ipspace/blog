PowerShell is a great scripting environment if your vendor provided PowerShell libraries to control their software or devices… but what if all you got is REST API (example: Nexus switches)?

We’ll conveniently ignore the challenges of managing devices that use 30-year-old non-scriptable CLI.

In the third part of his PowerShell for Networking Engineers Mitja Robas described how to use built-in PowerShell functionality to work with devices that have REST API, and showed how you can manage and configure Nexus OS switches with PowerShell. He also published the source code for all his examples on GitHub.
