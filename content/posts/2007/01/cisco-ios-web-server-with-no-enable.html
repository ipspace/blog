--- url: /2007/01/cisco-ios-web-server-with-no-enable.html title: "Cisco IOS web server with no enable password" date: "2007-01-31T11:05:00.000+01:00" tags: [ access control,web ] --- IOS has (yet another) nice "underdocumented" feature - if you don't have an enable password/secret configured on your router, you can access IOS HTTP(S) server (assuming it's enabled with the ip http server command) without any authentication whatsoever. Of course you'd never do that in a production environment, but it's nice to know you can always configure the router from a web browser if needed (see also the discussion on default passwords with Cisco SDM).
