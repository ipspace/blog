--- url: /2007/02/looking-for-good-embedded-event-manager.html title: "Looking for interesting Embedded Event Manager challenges" date: "2007-02-11T08:24:00.000+01:00" tags: [ EEM ] --- I've started collecting interesting EEM solutions, for example:
  • Automatic midnight upload of router configuration if it has been changed during the previous day (section Advanced deployment scenarios in the IP Corner article Keep Track of Router Configurations with Configuration Archive).
  • Report state changes in tracked objects (section Monitoring reliable static routing in the IP Corner article Small Site Multi-homing).
  • Report IP route appearance or disappearance.
  • React on IP SLA failures on out-of-bounds measurements.
  • Log console commands.
  • Periodic router reload.

If you have an interesting challenge that you think might be solvable with EEM, please let me know (use the Send a message to Ivan link in my bio page) ... after reading the following smallprint:
