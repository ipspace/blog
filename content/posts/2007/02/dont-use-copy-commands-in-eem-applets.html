--- url: /2007/02/dont-use-copy-commands-in-eem-applets.html title: "Don't use copy commands in EEM applets" date: "2007-02-01T13:58:00.000+01:00" tags: [ EEM ] --- I've tried to be a "good citizen" in one of my Embedded Eveng Manager (EEM) applications and used the action 1.1 cli command "copy running-config startup-config" instead of the (supposedly obsolete) action 1.1 cli command "write memory" command. However, as the copy command generates console prompts (unless you turn them off with the file prompt quiet command), it immediately hung my EEM applet.

Moral of the story: use write memory :)
